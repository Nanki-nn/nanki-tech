## 第一版优先文件驱动

个人站的第一版，最重要的是能持续更新。相比一开始就接 Headless CMS，TOML、Markdown 或 MDX 更适合轻量起步：内容和代码放在同一个仓库，改动可以被 Git 记录，也方便部署成静态页面。

这个站点目前用 TOML 管列表，用 Markdown 管长正文。TOML 适合标题、日期、标签、链接这类结构化字段；Markdown 适合文章、简介和长段落。

## 什么时候需要 MDX

如果文章里需要嵌入 React 组件，例如交互式图表、可运行 Demo 或复杂代码预览，MDX 会更合适。但 MDX 也会带来更多构建和内容约束，不一定适合每篇文章。

一个简单判断是：如果 90% 的内容只是文字、列表和链接，Markdown 就够了。只有当组件成为文章表达的一部分时，再升级到 MDX。

## 内容模型要稳定

个人站的内容系统不需要复杂，但字段要稳定。比如博客条目可以固定保留 `title`、`date`、`tags`、`slug` 和 `source`。这样列表页、详情页、SEO 元数据都能从同一份配置里读取。

稳定的内容模型还有一个好处：以后迁移到 CMS 时，也可以把这些字段直接映射过去，而不用重写整个前端。

## 静态导出更省心

如果没有登录、评论、搜索后台这类动态能力，静态导出通常是个人站最省心的部署方式。构建时生成 HTML，线上只需要托管静态文件，速度快，维护成本也低。
